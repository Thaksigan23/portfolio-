Building My Portfolio with Vanilla JS

2026-05-22 · Engineering

I chose to build this portfolio with plain HTML, CSS, and JavaScript instead of React or a static site generator. Here's why—and what I learned along the way.

Why vanilla?

  • GitHub Pages: Zero build step—just push and deploy
  • Performance: No framework bundle; the site loads fast on mobile
  • Control: Full control over animations, theming, and layout without fighting a framework
  • Demonstrates fundamentals: Shows I understand DOM APIs, CSS, and JS—not just frameworks

Features I built

  • Dark/light theme with localStorage persistence
  • Scroll-reveal animations with IntersectionObserver
  • A filterable journey timeline and project modal system
  • A code-wise blog powered by posts.json
  • Mobile drawer navigation and responsive layouts

Lessons learned

The biggest takeaway: cache-busting matters. When I updated the CSS, browsers kept serving old styles until I bumped the ?v= query string on asset links. I also learned that scroll-reveal animations need fallbacks—sections can stay invisible if the observer doesn't fire reliably on mobile.
