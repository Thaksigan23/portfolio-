Multi-Tenant SaaS with Supabase RLS

2026-06-28 · Engineering

When building Orbit Client Portal — a white-label Progressive Web App for B2B client collaboration — the non-negotiable requirement was data isolation. Each company that signs up gets its own workspace. Users from Company A must never see Company B's assets, messages, or workflows. Application-level filtering alone isn't enough; the database itself has to enforce boundaries.

Why Row-Level Security?

Supabase Row-Level Security (RLS) lets you define policies directly on PostgreSQL tables. Even if application code has a bug or a query forgets a WHERE org_id = ? clause, RLS blocks unauthorized rows at the database layer. For a multi-tenant B2B product, that's the safety net you want.

Designing workspace isolation

  • Every tenant-scoped table carries an organization_id foreign key
  • RLS policies compare organization_id against the authenticated user's org claim
  • SQL triggers auto-assign org context on insert so new records inherit the correct workspace
  • Admin roles within an org get broader read/write policies without crossing org boundaries

Next.js App Router on top

The frontend uses Next.js App Router for server-side rendering, dynamic routing, and optimized data-fetching. SSR matters here because client portals need to feel instant — a slow first paint undermines trust in a product meant for professional collaboration. Server components fetch tenant-scoped data through Supabase's server client, and the RLS policies apply regardless of whether the query runs on the server or client.

Encrypted Asset Vault

Sensitive client files live in cloud storage buckets with time-limited signed URLs. The portal never exposes permanent public links. When a user requests a file, the backend generates a short-lived URL scoped to their organization and role. Combined with RLS on the metadata table, this gives defense in depth for file access.

Key takeaways

  • Multi-tenancy is a database design problem first, a UI problem second
  • RLS + triggers reduce the blast radius of application bugs
  • White-label PWAs need fast SSR and strict tenant scoping to feel enterprise-grade
  • Signed URLs are essential when storing sensitive files outside the primary database
